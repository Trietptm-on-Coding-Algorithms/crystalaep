Alerts Browser

This help section provides an overview of how to operate the Alerts Browser feature.

Repository/Sorting Bar

The Alerts Browser offers a means of viewing all the alerts which have been raised in response to threats prevented by Crystal. The view is similar to that offered by the main user interface but differs in that alerts can be viewed by repository (category) and can be sorted (for example by Risk or Date). The threat respositories which are available to be viewed are as follows:

All - Threats from all categories are displayed in the alerts browser.

AntiSpray - Threats blocked as a result of anti-heap spray mitigations are displayed in the alerts browser. The issues listed under this repository are always high risk.

ApiFirewall - Threats blocked as a result of violating the API firewall are displayed in the alerts browser. The issues listed under this repository are generally medium to high risk.

AudioValidator - Threats involving invalid audio content are displayed in the alerts browser. Crystal uses this repository at present to log instances of broken audio (which can occasionally indicate an attempt to exploit a security flaw). These threats are generally low risk.

ConnectionMonitor - Blocked HTTP connections are displayed in the alerts browser. HTTP connections are blocked as a result of filter modules (which scan and block or modify Internet traffic) determining that data is not safe for processing. Threats listed here can be low to high risk.

DfnsInDepth - Threats blocked as a result of violating Defense-in-Depth mechanisms configured to help block the execution of malicious code. The threats listed here vary and are generally high risk.

HeapMonitor - Threats involving corruption or abuse of application heap memory are displayed in the alerts browser. Threats detailed here are almost always high risk.

ImageValidator - Threats involving invalid image content are displayed in the alerts browser. Crystal uses this repository at present to log instances of broken images (which can occasionally indicate an attempt to exploit a security flaw). These threats are generally low risk.

StackMonitor - Threats involving corruption or abuse of application stack memory are displayed in the alerts browser. Threats detailed here are almost always high risk.

VideoValidator - Threats involving invalid video content are displayed in the alerts browser. Crystal uses this repository at present to log instances of broken videos (which can occasionally indicate an attempt to exploit a security flaw). These threats are generally low risk.

WebAppMonitor - Threats blocked as a result of attempts to exploit vulnerabilities in web-sites to obtain sensitive data are displayed in the alerts browser. These issues are generally medium risk as they cannot typically lead to malicious code being installed, but may lead to financial or other data compromise.

Alerts displayed can be sorted by the following methods:

Date - Alerts are sorted by date from most recent to least (default display).

Date (Reverse) - Alerts are sorted with least recent displayed first.

Risk - Alerts are sorted by risk with most high risk displayed first.

Risk (Reverse) - Alerts are sorted by risk with least high risk displayed first.

Process Name - Alerts are sorted by process name (application name).

Additionally the number of items to display can be configured (the default setting is 500).
